Variantes d'Animation

Les indicateurs de chargement proposent plusieurs styles d'animation pour s'adapter à différents contextes d'utilisation et préférences de design.

Circle

Animation circulaire classique et reconnaissable, idéale pour la plupart des contextes de chargement.

Dots

Points rebondissants qui créent une animation rythmique et ludique, parfaite pour les interfaces modernes.

Pulse

Barres pulsantes qui créent un effet de progression visuelle, idéal pour les processus de chargement séquentiels.

Couleurs Contextuelles

Les indicateurs peuvent adopter différentes couleurs pour communiquer visuellement le type de processus en cours ou son état.

Primary

color="primary"

Utilisé pour les actions principales et les chargements standard de l'application.

Secondary

color="secondary"

Pour les actions et processus secondaires, moins prioritaires dans l'interface.

Success

color="success"

Indique un chargement lié à une opération réussie ou en cours de validation.

Warning

color="warning"

Utilisé pour les processus nécessitant une attention particulière de l'utilisateur.

Danger

color="danger"

Pour les processus critiques ou les rechargements suite à une erreur.

Tailles Disponibles

Trois tailles sont disponibles pour s'adapter à différents contextes d'utilisation, des petits espaces aux zones de chargement principales.

Small

24px de diamètre size="small"

Idéal pour les espaces réduits, les boutons, ou les indicateurs de chargement inline dans le texte.

Medium (Défaut)

32px de diamètre size="medium"

Taille par défaut, équilibrée pour la plupart des cas d'usage et zones de chargement standards.

Large

48px de diamètre size="large"

Pour les chargements de page complète ou les zones nécessitant une forte visibilité de l'indicateur.

Exemples de Combinaisons

Découvrez comment combiner les différentes variantes, couleurs et tailles pour créer des indicateurs adaptés à vos besoins spécifiques.

Dots + Success + Large

Parfait pour confirmer visuellement qu'un processus de validation est en cours.

Pulse + Warning + Medium

Utilisé pour des processus nécessitant l'attention de l'utilisateur pendant le chargement.

Circle + Secondary + Small

Idéal pour les processus discrets en arrière-plan ou dans des interfaces denses.

Documentation API

Le composant SpinLoader utilise des input signals pour une API réactive et performante, conforme aux standards Angular 20.

Propriétés (Input Signals)

Propriété Type Défaut Description
variant SpinLoaderVariant 'circle' 'circle' | 'dots' | 'pulse' - Style d'animation de l'indicateur
size SpinLoaderSize 'medium' 'small' | 'medium' | 'large' - Taille de l'indicateur
color SpinLoaderColor 'primary' 'primary' | 'secondary' | 'success' | 'warning' | 'danger' - Couleur contextuelle
label string | undefined undefined Label textuel optionnel affiché sous l'indicateur
ariaLabel string 'Chargement en cours' Label ARIA pour l'accessibilité, annoncé par les lecteurs d'écran
ariaLive 'polite' | 'assertive' 'polite' Niveau de politesse pour les lecteurs d'écran (polite pour standard, assertive pour urgent)

Exemple d'Utilisation Complète

import { PshSpinLoaderComponent } from 'ps-helix';

@Component({
  standalone: true,
  imports: [PshSpinLoaderComponent],
  template: `
    <div class="loading-container">
      <psh-spinloader
        variant="circle"
        color="primary"
        size="large"
        ariaLabel="Chargement des données utilisateur"
        ariaLive="polite"
      ></psh-spinloader>
      <p>Chargement en cours...</p>
    </div>
  `
})
export class MyComponent {}

Bonnes Pratiques d'Accessibilité

  • ARIA Label : Toujours fournir un ariaLabel descriptif qui explique clairement ce qui est en cours de chargement
  • ARIA Live : Utilisez ariaLive="polite" pour les chargements standards et ariaLive="assertive" uniquement pour les chargements critiques
  • Role Status : Le composant définit automatiquement role="status" et aria-busy="true" pour une accessibilité optimale
  • Mouvement Réduit : Le composant détecte automatiquement la préférence prefers-reduced-motion et adapte les animations en conséquence
  • Contexte Visuel : Accompagnez toujours l'indicateur d'un texte explicatif pour les utilisateurs qui ne peuvent pas voir l'animation
  • Feedback Temps Réel : Pour les processus longs, envisagez d'ajouter un label dynamique avec la progression ou l'étape en cours

Configuration Globale

Vous pouvez configurer les valeurs par défaut globalement en fournissant SPINLOADER_CONFIG :

import { SPINLOADER_CONFIG } from 'ps-helix';

bootstrapApplication(AppComponent, {
  providers: [
    {
      provide: SPINLOADER_CONFIG,
      useValue: {
        variant: 'dots',
        size: 'medium',
        color: 'primary'
      }
    }
  ]
});
