Fonctionnement

  • Navigation linéaire : Les étapes doivent être complétées séquentiellement
  • Clic sur étape complétée : Permet de revenir consulter une étape précédente
  • Clavier : Flèches gauche/droite, Home, End, Enter, Espace
  • Accessibilité : Roles ARIA complets (tablist/tab, aria-selected, aria-current)

Documentation API

Propriétés State Flow Indicator

Propriété Type Défaut Description
activeStep number 0 Index de l'étape active (two-way binding)
linear boolean true Force la progression séquentielle
size 'small' | 'medium' | 'large' 'medium' Taille des segments
beforeStepChange (from, to) => boolean | Promise<boolean> undefined Validation avant changement d'étape

Propriétés Flow Step

Propriété Type Défaut Description
title string required Nom de l'étape affiché dans le segment
completed boolean false Marque l'étape comme validée
disabled boolean false Désactive l'étape
loading boolean false Affiche un spinner dans le segment
error string undefined Active l'état erreur (segment rouge)
warning string undefined Active l'état warning (segment orange)

Événements

Événement Type Description
stepChange number Émis lors du changement d'étape
completed void Émis quand la dernière étape est atteinte et complétée
navigationError string Émis lors d'une erreur de navigation

Méthodes Publiques

Méthode Signature Description
next() Promise<void> Passe à l'étape suivante
previous() Promise<void> Revient à l'étape précédente
goToStep(index) Promise<void> Navigation directe vers une étape
reset() void Réinitialise à l'étape 0
