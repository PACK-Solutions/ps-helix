Apparences de Boutons

Les boutons offrent quatre apparences distinctes pour s'adapter à différents contextes d'utilisation et niveaux de hiérarchie visuelle.

Filled Button

Filled

Style par défaut avec fond plein. Idéal pour les actions principales et les CTA qui nécessitent une forte visibilité.

Outline Button

Outline

Style avec bordure sans fond. Parfait pour les actions secondaires et les interfaces épurées nécessitant moins de poids visuel.

Rounded Button

Rounded

Style avec coins complètement arrondis. Adapté aux designs modernes et interfaces ludiques nécessitant une touche distinctive.

Text Button

Text

Style minimaliste sans fond ni bordure. Utilisé pour les actions tertiaires, la navigation et les menus où la discrétion est nécessaire.

Variantes de Boutons

Chaque apparence supporte cinq variantes de couleur pour communiquer différents niveaux d'importance et de sémantique adaptés à chaque action.

Primary Primary Primary Primary

Primary

Actions principales de l'application, CTA et conversions. Utilisé avec parcimonie pour guider l'utilisateur vers l'action la plus importante.

Secondary Secondary Secondary Secondary

Secondary

Actions secondaires et alternatives. Offre une option complémentaire sans rivaliser avec l'action principale.

Success Success Success Success

Success

Actions positives et confirmations. Utilisé pour valider des modifications, créer de nouveaux éléments ou confirmer des actions bénéfiques.

Warning Warning Warning Warning

Warning

Actions nécessitant attention ou prudence. Alerte l'utilisateur sur des conséquences potentielles sans être destructif.

Danger Danger Danger Danger

Danger

Actions destructives et irréversibles. Réservé aux suppressions, annulations définitives et actions à haut risque nécessitant confirmation.

Tailles Disponibles

Trois tailles sont disponibles pour s'adapter à différents contextes d'utilisation et densités d'interface.

Petit Bouton

Small

32px de hauteur size="small"

Idéal pour les interfaces denses, les tableaux de données ou les actions compactes nécessitant peu d'espace.

Bouton Moyen

Medium (Défaut)

40px de hauteur size="medium"

Taille par défaut, équilibrée pour la plupart des cas d'usage et une bonne lisibilité sur tous les appareils.

Grand Bouton

Large

48px de hauteur size="large"

Pour les CTA importants, les interfaces mobiles ou les actions nécessitant une grande visibilité et une accessibilité tactile optimale.

Boutons avec Icônes

Les icônes renforcent la compréhension visuelle des actions et améliorent la reconnaissance. Utilisez des icônes pertinentes pour chaque contexte.

Icône à gauche

Icône à Gauche (Défaut)

iconPosition="left"

Position standard pour la plupart des actions. L'icône précède le texte dans l'ordre de lecture.

Icône à droite

Icône à Droite

iconPosition="right"

Utilisé pour les actions de navigation forward, les téléchargements ou les actions qui suggèrent un mouvement vers la droite.

Icône Seule

iconPosition="only"

Bouton circulaire avec icône uniquement. Nécessite toujours un iconOnlyText descriptif pour l'accessibilité.

Valider Supprimer Télécharger

Icônes Contextuelles

Sélectionnez des icônes qui renforcent visuellement l'action décrite. Les icônes doivent être immédiatement reconnaissables.

  • check pour valider ou confirmer
  • trash pour supprimer
  • download pour télécharger
  • plus pour créer ou ajouter
  • arrow-* pour naviguer

États des Boutons

Les boutons supportent différents états pour communiquer clairement leur disponibilité et leur statut d'exécution.

État Normal

Bouton normal Bouton normal

État par défaut du bouton, prêt à recevoir les interactions utilisateur.

État Désactivé

Bouton désactivé Bouton désactivé

Action temporairement indisponible. Le bouton n'accepte aucune interaction et communique visuellement son indisponibilité.

État de Chargement

Chargement Chargement

Action en cours d'exécution. Un indicateur de chargement remplace temporairement le contenu pour communiquer l'état de progression.

Bouton Pleine Largeur

Bouton pleine largeur

S'étend sur toute la largeur du conteneur parent. Idéal pour les formulaires et les actions principales sur mobile.

Caractéristiques des États

  • Hover : Changement visuel au survol pour indiquer l'interactivité
  • Focus : Indicateur de focus visible pour la navigation au clavier
  • Active : Feedback visuel lors du clic pour confirmer l'interaction
  • Disabled : Curseur not-allowed et opacité réduite
  • Loading : Spinloader animé avec aria-busy="true"

Exemple d'Utilisation Complète

import { PshButtonComponent } from 'ps-helix';

@Component({
  imports: [PshButtonComponent],
  template: `
    <psh-button
      appearance="filled"
      variant="primary"
      size="medium"
      type="submit"
      icon="arrow-right"
      iconPosition="right"
      [disabled]="isFormInvalid"
      [loading]="isSubmitting"
      [fullWidth]="false"
      loadingText="Envoi en cours..."
      (clicked)="handleSubmit($event)"
    >
      Continuer
    </psh-button>
  `
})
export class MyComponent {
  isFormInvalid = false;
  isSubmitting = false;

  handleSubmit(event: MouseEvent): void {
    this.isSubmitting = true;
  }
}

Bonnes Pratiques d'Accessibilité

  • Labels descriptifs : Utilisez un texte clair décrivant l'action (évitez "Cliquez ici" ou "En savoir plus" sans contexte)
  • Attribut iconOnlyText : Obligatoire pour les boutons avec iconPosition="only" pour communiquer l'action aux lecteurs d'écran
  • États visuels : Maintenez un indicateur de focus visible avec un ratio de contraste minimum de 3:1 selon WCAG 2.1
  • Support clavier : Les boutons sont automatiquement accessibles via Tab (navigation), Enter et Space (activation)
  • Feedback d'état : Utilisez aria-busy="true" pendant le chargement et aria-disabled="true" pour l'état désactivé
  • Hiérarchie visuelle : Limitez l'utilisation de la variante primary à une seule action principale par vue pour guider clairement l'utilisateur
  • Contraste : Vérifiez que le ratio de contraste entre le texte et le fond respecte au minimum 4.5:1 (WCAG AA) pour tous les états
