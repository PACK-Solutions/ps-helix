Options de Configuration

Les modales supportent diverses options pour contrôler leur comportement et personnaliser l'expérience utilisateur.

Fermeture sur l'Arrière-plan

Contrôle si la modale se ferme en cliquant sur l'arrière-plan sombre.

Modale sans fermeture backdrop

Cette modale ne peut être fermée qu'en cliquant sur le bouton de fermeture ou les boutons d'action.

Le clic sur l'arrière-plan et la touche Échap sont désactivés.

Bouton de Fermeture

Active ou désactive l'affichage du bouton de fermeture dans l'en-tête.

Prévention du Scroll

Empêche le défilement de la page en arrière-plan lorsque la modale est ouverte.

Caractéristiques des Options

  • closeOnBackdrop : Ferme la modale au clic sur l'arrière-plan (défaut: true)
  • closeOnEscape : Ferme la modale avec la touche Échap (défaut: true)
  • showClose : Affiche le bouton de fermeture (défaut: true)
  • preventScroll : Empêche le scroll de la page (défaut: true)
  • showFooter : Affiche le footer par défaut (défaut: true)

Documentation API

Le composant Modal utilise des model inputs et des input signals pour une API simple et performante, conforme aux standards Angular 20.

Propriétés (Model Inputs & Signals)

Propriété Type Défaut Description
open boolean false Contrôle la visibilité de la modale (two-way binding)
title string 'Modal Title' Titre affiché dans l'en-tête de la modale
size ModalSize 'medium' 'small' | 'medium' | 'large' - Taille de la modale
showClose boolean true Affiche le bouton de fermeture dans l'en-tête
closeOnBackdrop boolean true Ferme la modale au clic sur l'arrière-plan
closeOnEscape boolean true Ferme la modale avec la touche Échap
preventScroll boolean true Empêche le défilement de la page en arrière-plan
showFooter boolean true Affiche le footer par défaut avec boutons d'action
dismissLabel string 'Close' Label accessible pour le bouton de fermeture
confirmLabel string 'Confirm' Label du bouton de confirmation dans le footer
cancelLabel string 'Cancel' Label du bouton d'annulation dans le footer
styleClass string '' Classes CSS personnalisées à appliquer au conteneur de la modale

Signaux Calculés (Computed Signals)

Signal Type Description
isMobileScreen Signal<boolean> Indique si l'écran est de taille mobile (breakpoint: 768px). Utile pour adapter les footers personnalisés.

Événements (Outputs)

Événement Type Description
closed OutputEmitterRef<void> Émis lorsque la modale est fermée
confirmed OutputEmitterRef<void> Émis lors du clic sur le bouton de confirmation
openChange OutputEmitterRef<boolean> Émis automatiquement par le model() lors du changement de l'état open

Slots de Contenu (Content Projection)

Sélecteur Description
[modal-title] Permet de personnaliser le titre avec du HTML ou des composants (ex: titre avec icône)
[modal-footer] Remplace le footer par défaut. Nécessite #modalFooter pour la détection automatique.
(contenu par défaut) Le contenu principal de la modale, projeté dans la zone de contenu scrollable

Exemple d'Utilisation Complète

import { PshModalComponent, PshButtonComponent } from 'ps-helix';

@Component({
  standalone: true,
  imports: [PshModalComponent, PshButtonComponent],
  template: `
    <psh-button (clicked)="isModalOpen = true">
      Ouvrir la modale
    </psh-button>

    <!-- Modale avec footer par défaut -->
    <psh-modal
      [(open)]="isModalOpen"
      title="Confirmer l'action"
      size="medium"
      [closeOnBackdrop]="true"
      [closeOnEscape]="true"
      [showClose]="true"
      [showFooter]="true"
      confirmLabel="Confirmer"
      cancelLabel="Annuler"
      (confirmed)="handleConfirm()"
      (closed)="handleClose()"
    >
      <p>Êtes-vous sûr de vouloir continuer ?</p>
    </psh-modal>

    <!-- Modale avec footer personnalisé responsive -->
    <psh-modal #customModal [(open)]="isCustomModalOpen" title="Actions Multiples">
      <p>Choisissez une action à effectuer.</p>
      <div modal-footer #modalFooter>
        <psh-button
          appearance="text"
          [fullWidth]="customModal.isMobileScreen()"
          (clicked)="isCustomModalOpen = false"
        >
          Annuler
        </psh-button>
        <psh-button
          appearance="filled"
          variant="success"
          [fullWidth]="customModal.isMobileScreen()"
          (clicked)="handleSave()"
        >
          Enregistrer
        </psh-button>
        <psh-button
          appearance="filled"
          variant="primary"
          [fullWidth]="customModal.isMobileScreen()"
          (clicked)="handlePublish()"
        >
          Publier
        </psh-button>
      </div>
    </psh-modal>
  `
})
export class MyComponent {
  isModalOpen = false;
  isCustomModalOpen = false;

  handleConfirm(): void {
    console.log('Action confirmée');
    this.isModalOpen = false;
  }

  handleClose(): void {
    console.log('Modale fermée');
  }

  handleSave(): void {
    console.log('Enregistrement...');
    this.isCustomModalOpen = false;
  }

  handlePublish(): void {
    console.log('Publication...');
    this.isCustomModalOpen = false;
  }
}

Bonnes Pratiques d'Accessibilité

  • Focus Management : Le focus est automatiquement placé sur le premier élément focusable à l'ouverture et restauré à la fermeture
  • Focus Trap : La navigation au clavier (Tab) reste confinée dans la modale, empêchant l'accès aux éléments en arrière-plan
  • Attributs ARIA : La modale utilise role="dialog", aria-labelledby pour le titre et aria-describedby pour le contenu
  • Touche Échap : Permet de fermer la modale rapidement (désactivable si nécessaire pour les confirmations critiques)
  • Scroll Lock : La page en arrière-plan est verrouillée pour éviter les conflits de navigation et maintenir l'attention
  • Backdrop Sémantique : L'arrière-plan sombre indique visuellement que le contenu est modal et nécessite une action
  • Labels Descriptifs : Utilisez toujours des titres clairs et des labels de boutons explicites décrivant l'action
  • Utilisation Modérée : Évitez les modales imbriquées et limitez leur usage aux interactions nécessitant vraiment l'attention complète
  • Footer Personnalisé Responsive : Pour les footers personnalisés, utilisez toujours [fullWidth]="modal.isMobileScreen()" sur vos boutons pour garantir une expérience mobile cohérente (breakpoint: 768px)

Configuration Globale (MODAL_CONFIG)

Utilisez le token MODAL_CONFIG pour définir les valeurs par défaut de toutes les modales de l'application.

import { MODAL_CONFIG } from 'ps-helix';

providers: [
  {
    provide: MODAL_CONFIG,
    useValue: {
      size: 'medium',
      showClose: true,
      closeOnBackdrop: true,
      closeOnEscape: true,
      preventScroll: true,
      showFooter: true,
      dismissLabel: 'Fermer',
      confirmLabel: 'Confirmer',
      cancelLabel: 'Annuler'
    }
  }
]

Service ModalService

Le ModalService permet de gérer les instances de modales et de suivre leur état global.

Membre Type Description
activeModalsCount Signal<number> Signal calculé indiquant le nombre de modales actuellement ouvertes
getConfig() Partial<ModalConfig> Retourne la configuration globale des modales
isRegistered(id) boolean Vérifie si une modale avec l'ID donné est actuellement ouverte 
import { ModalService } from 'ps-helix';

@Component({ ... })
export class MyComponent {
  private modalService = inject(ModalService);

  get hasOpenModals(): boolean {
    return this.modalService.activeModalsCount() > 0;
  }
}

Gestion du Scroll Lock

Lorsqu'une modale est ouverte avec preventScroll="true", le composant ajoute la classe modal-open au body et calcule la largeur de la scrollbar pour éviter les décalages de mise en page.

body.modal-open {
  overflow: hidden;
  padding-right: var(--scrollbar-width);
}
