États des Cartes

Les cartes supportent différents états pour communiquer clairement leur disponibilité et leur statut d'exécution.

État Interactif

Cette carte est cliquable avec effet de survol et focus accessible au clavier.

Carte cliquable avec effet de survol et support complet de la navigation au clavier pour une accessibilité optimale.

État de Chargement

Ce contenu ne sera pas visible pendant le chargement

Affiche un skeleton animé pendant le chargement des données asynchrones.

État Désactivé

Cette carte est désactivée avec opacité réduite et interactions bloquées.

Action impossible

Carte temporairement indisponible avec opacité réduite et interactions bloquées.

Caractéristiques des États

  • Interactive : Rend la carte cliquable avec l'événement clicked
  • Hoverable : Ajoute un effet de survol avec élévation et transition fluide
  • Loading : Affiche un skeleton animé avec aria-busy="true"
  • Disabled : Réduit l'opacité à 0.6 et bloque toutes les interactions
  • Focus : Indicateur de focus visible pour la navigation au clavier

  [hoverable]="true"

  [disabled]="isDisabled"

>
  Contenu
</psh-card>

Documentation API

Le composant Card utilise des model inputs et des input signals pour une API simple et performante, conforme aux standards Angular 20.

Propriétés (Model Inputs)

Propriété Type Défaut Description
variant CardVariant 'default' 'default' | 'elevated' | 'outlined'
colorVariant CardColorVariant 'default' 'default' | 'info' | 'success' | 'warning' | 'danger'
density CardDensity 'normal' 'compact' | 'normal' | 'spacious'
interactive boolean false Rend la carte cliquable
hoverable boolean false Ajoute un effet de survol
loading boolean false Affiche un skeleton de chargement
disabled boolean false Désactive la carte

Propriétés (Input Signals)

Propriété Type Défaut Description
title string '' Titre principal de la carte
description string '' Description ou sous-titre
ariaLabel string | undefined undefined Label ARIA personnalisé pour l'accessibilité

Événements (Outputs)

Événement Type Description
clicked EventEmitter<MouseEvent | KeyboardEvent> Émis lors du clic sur la carte (si interactive)

Slots Disponibles

Slot Description
card-header-icon Icône affichée avant le titre dans le header
card-header-content Contenu personnalisé pour remplacer le header par défaut
card-header-extra Badge ou actions secondaires dans le header (à droite)
(default) Contenu principal de la carte (body)
card-footer Métadonnées ou informations complémentaires en bas de carte
card-actions Boutons d'action avec alignement automatique

Exemple d'Utilisation Complète

import { PshCardComponent, PshButtonComponent } from 'ps-helix';

@Component({
  standalone: true,
  imports: [PshCardComponent, PshButtonComponent],
  template: `
    <psh-card
      title="Mon Projet"
      description="Description du projet"
      variant="elevated"
      colorVariant="default"
      density="normal"
      [interactive]="true"
      [hoverable]="true"
      [loading]="isLoading"
      [disabled]="isDisabled"
      ariaLabel="Carte de projet interactive"
      (clicked)="handleCardClick($event)"
    >
      <div card-header-icon>
        <i class="ph ph-folder"></i>
      </div>
      <div card-header-extra>
        <psh-tag variant="success">Actif</psh-tag>
      </div>

      <p>Contenu principal de la carte avec description détaillée.</p>

      <div card-footer>
        <span>Échéance: 31 Déc 2025</span>
        <span>Équipe: 8 personnes</span>
      </div>

      <div card-actions>
        <psh-button variant="primary">Ouvrir</psh-button>
        <psh-button appearance="outline" variant="secondary">Détails</psh-button>
      </div>
    </psh-card>
  `
})
export class MyComponent {
  isLoading = false;
  isDisabled = false;

  handleCardClick(event: MouseEvent | KeyboardEvent): void {
    console.log('Card clicked', event);
  }
}

Bonnes Pratiques

  • Titre et description : Utilisez toujours title et description pour un rendu cohérent et accessible
  • Densité par défaut : Privilégiez density="normal" pour la plupart des cas d'usage
  • Variantes de couleur : Réservez les colorVariant (info, success, warning, danger) pour les messages d'état importants
  • Interactivité : Ajoutez [interactive]="true" uniquement si la carte doit être cliquable dans son ensemble
  • Effet de survol : Combinez [hoverable]="true" avec interactive pour un feedback visuel clair
  • État de chargement : Utilisez [loading]="true" pendant les opérations asynchrones pour améliorer l'expérience utilisateur
  • Accessibilité : Fournissez un ariaLabel descriptif pour les cartes interactives
  • Actions : Placez les boutons principaux dans le slot card-actions pour un alignement automatique et cohérent. Les boutons deviennent automatiquement pleine largeur sur mobile (< 640px) grâce au CSS intégré
