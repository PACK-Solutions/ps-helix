Variantes de Toasts

Les toasts offrent différentes variantes visuelles pour communiquer des niveaux d'importance et de contexte adaptés à chaque situation.

Afficher Toast Info

Info

Messages informatifs généraux, mises à jour système et notifications non critiques qui apportent un contexte ou une aide.

Afficher Toast Success

Success

Confirmations d'actions réussies, processus terminés avec succès et validations positives.

Afficher Toast Warning

Warning

Avertissements et précautions nécessitant l'attention de l'utilisateur sans être bloquants.

Afficher Toast Danger

Danger

Erreurs critiques, échecs d'opération et problèmes nécessitant une action immédiate de l'utilisateur.

Positions Disponibles

  • Top Right (Défaut) : Position standard pour la plupart des applications, n'interfère pas avec la navigation
  • Top Left : Alternative pour les interfaces RTL ou les applications avec navigation à droite
  • Bottom Right : Évite les conflits avec le header, idéal pour les interfaces avec navigation supérieure
  • Bottom Left : Pour des cas d'usage spécifiques ou des interfaces personnalisées

Fonctionnalités Avancées

Les toasts supportent des fonctionnalités avancées pour améliorer l'expérience utilisateur et s'adapter à des besoins complexes.

Toasts Multiples

Empilage de Toasts

Les toasts s'empilent automatiquement lorsque plusieurs notifications sont affichées simultanément. La limite par défaut est de 5 toasts visibles.

  • Empilage automatique des notifications
  • Gestion FIFO (First In, First Out)
  • Limite configurable de toasts visibles
  • Animation de pile fluide
Pause

Pause

Les toasts pausent automatiquement leur timer de fermeture lorsque l'utilisateur les survole, permettant une lecture sans interruption.

  • Pause automatique au survol de la souris
  • Reprise du timer après le survol
  • Améliore l'accessibilité et l'UX
  • Configurable globalement

Contrôle de Fermeture

Le système de toast offre une flexibilité complète pour contrôler comment les utilisateurs peuvent fermer les notifications. Chaque toast peut être configuré individuellement pour s'adapter au contexte et à l'importance du message.

Toast Standard

Configuration Standard (Défaut)

duration: 5000, showCloseButton: true

Le toast se ferme automatiquement après 5 secondes, mais l'utilisateur peut le fermer manuellement à tout moment avec le bouton de fermeture.

  • Fermeture automatique après expiration du délai
  • Fermeture manuelle via bouton croix disponible
  • Pause au survol pour lecture sans pression
  • Cas d'usage : notifications générales, confirmations, messages informatifs
Fermeture Auto Uniquement

Fermeture Automatique Uniquement

duration: 5000, showCloseButton: false

Le toast se ferme automatiquement après le délai spécifié. Aucun bouton de fermeture manuelle n'est affiché, garantissant que l'utilisateur voit le message pendant toute sa durée.

  • Fermeture automatique après expiration du délai
  • Pas de fermeture manuelle pour garantir la lecture
  • Durée recommandée : 3 à 7 secondes maximum
  • Cas d'usage : messages importants à lire obligatoirement, confirmations critiques, alertes de sécurité courtes
Fermeture Manuelle Obligatoire

Fermeture Manuelle Obligatoire

duration: 0, showCloseButton: true

Le toast persiste jusqu'à ce que l'utilisateur le ferme manuellement. Idéal pour les actions critiques nécessitant une confirmation explicite de lecture.

  • Pas de fermeture automatique, le toast reste visible
  • Fermeture manuelle obligatoire via bouton croix
  • Attention requise de l'utilisateur
  • Cas d'usage : erreurs critiques, actions nécessitant confirmation, messages d'avertissement importants

Recommandations d'Utilisation

À Faire
  • Utilisez la fermeture automatique pour les messages généraux et notifications non critiques
  • Réservez la fermeture manuelle obligatoire aux erreurs critiques et actions importantes
  • Limitez la durée des toasts sans bouton de fermeture (max 7 secondes)
  • Utilisez des messages clairs et concis pour tous les types de toast
À Éviter
  • Ne créez JAMAIS de toast persistant sans bouton de fermeture (duration: 0, showCloseButton: false)
  • N'abusez pas des toasts persistants, ils peuvent surcharger l'interface
  • Évitez les durées trop longues (>10s) pour les toasts auto-fermables
  • Ne masquez pas le bouton de fermeture sur des messages longs ou complexes

Installation et Configuration

// 1. Importer le composant Toast dans app.component.ts
import { PshToastComponent } from 'ps-helix';

@Component({
  imports: [PshToastComponent],
  template: `
    <router-outlet />
    <psh-toast />
  `
})
export class AppComponent {}

// 2. Injecter le service dans votre composant
import { PshToastService } from 'ps-helix';

export class MyComponent {
  private toastService = inject(PshToastService);

  showNotification(): void {
    this.toastService.show({
      message: 'Opération réussie',
      type: 'success',
      duration: 3000,
      icon: 'check-circle'
    });
  }
}

Exemples d'Utilisation Avancée

// Toast persistant nécessitant une action
showCriticalError(): void {
  this.toastService.show({
    message: 'Erreur critique détectée',
    type: 'danger',
    duration: 0,
    showCloseButton: true,
    icon: 'warning-octagon'
  });
}

// Toast auto-fermable sans bouton de fermeture
showImportantMessage(): void {
  this.toastService.show({
    message: 'Message important à lire',
    type: 'warning',
    duration: 5000,
    showCloseButton: false
  });
}

// Toast avec gestion de l'ID
uploadFile(): void {
  const toastId = this.toastService.show({
    message: 'Téléchargement en cours...',
    type: 'info',
    duration: 0,
    showCloseButton: true
  });

  this.uploadService.upload().subscribe({
    next: () => {
      this.toastService.remove(toastId);
      this.toastService.show({
        message: 'Téléchargement terminé',
        type: 'success',
        duration: 3000
      });
    }
  });
}

Bonnes Pratiques d'Accessibilité

  • Rôle ARIA : Les toasts utilisent role="alert" et aria-live="polite" pour être annoncés par les lecteurs d'écran sans interrompre
  • Messages clairs : Rédigez des messages concis et explicites décrivant l'action ou l'état (évitez les messages ambigus)
  • Durée adaptée : Assurez une durée suffisante pour que tous les utilisateurs puissent lire le message (minimum 3 secondes pour les messages courts)
  • Contraste : Vérifiez que le ratio de contraste entre le texte et le fond respecte WCAG AA (4.5:1 minimum)
  • Fermeture accessible : Le bouton de fermeture est accessible au clavier via Tab et Enter, et la touche Escape ferme le toast actif
  • Éviter la surcharge : Limitez le nombre de toasts simultanés pour éviter de surcharger l'interface et l'utilisateur
  • Types appropriés : Utilisez le type correct (danger pour les erreurs critiques, info pour les notifications générales)
  • Pause au survol : Activez pauseOnHover pour permettre aux utilisateurs de lire les messages sans pression temporelle

Quand Utiliser les Toasts

  • Utilisez les toasts pour : Confirmations d'actions, notifications non-bloquantes, mises à jour de statut, feedback temporaire
  • N'utilisez PAS les toasts pour : Erreurs critiques bloquantes (utilisez des modales), informations permanentes (utilisez des alertes), contenu nécessitant une action immédiate
  • Alternatives : Pour des messages permanents ou critiques, préférez les alertes (psh-alert). Pour des confirmations nécessitant une action, utilisez des modales (psh-modal)
