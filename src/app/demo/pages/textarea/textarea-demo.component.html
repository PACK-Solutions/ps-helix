Variantes

Deux variantes distinctes s'adaptent à différents contextes d'interface et niveaux de densité visuelle.

Outlined

Style par défaut avec bordure prononcée. Idéal pour les formulaires où une délimitation claire des champs est nécessaire.

Filled

Style avec fond coloré. Parfait pour les interfaces denses et les formulaires modernes nécessitant une apparence plus subtile.

Tailles Disponibles

Trois tailles pour s'adapter à différents contextes d'utilisation et densités d'interface.

Small

size="small"

Pour les interfaces denses, tableaux ou formulaires compacts nécessitant peu d'espace vertical.

Medium (Défaut)

size="medium"

Taille équilibrée, adaptée à la majorité des cas d'usage avec une bonne lisibilité.

Large

size="large"

Pour les formulaires importants, interfaces mobiles ou zones de texte nécessitant une grande visibilité.

États

Le textarea supporte différents états pour communiquer clairement sa disponibilité et son statut de validation.

État Désactivé

Champ temporairement non modifiable. Utilisé lors d'une soumission ou d'un formulaire incomplet.

État Lecture Seule

Contenu visible et sélectionnable mais non modifiable. Permet de copier sans éditer.

État Erreur

Indique une validation échouée avec un message clair et actionnable (role="alert").

État Requis

Ajoute un astérisque visuel et aria-required="true" pour les lecteurs d'écran.

Caractéristiques des États

  • Hover : Bordure surlignée pour signaler l'interactivité
  • Focus : Anneau coloré visible pour la navigation au clavier
  • Disabled : Curseur not-allowed, opacité réduite
  • Readonly : Curseur default, fond distinct
  • Error : Bordure rouge, message d'erreur avec aria-invalid="true"
  • Required : Astérisque et aria-required="true"

Largeur Complète

La propriété fullWidth supprime la largeur maximale par défaut (480px) pour étendre le champ sur toute la largeur de son conteneur, idéal pour les descriptions longues ou les éditeurs de contenu.

Compteur de Caractères

Combinez maxLength et showCharacterCount pour afficher un compteur en temps réel. Le compteur passe en erreur au dépassement et est annoncé via aria-live="polite".

Comportement

  • Format : Affichage 0 / 200 quand maxLength est défini
  • Sans limite : Affichage simple du nombre courant
  • Dépassement : Passage en couleur erreur et aria-invalid
  • A11y : aria-live="polite" pour l'annonce en direct

Auto-dimensionnement

Avec autoSize, un effect() réactif ajuste automatiquement la hauteur du textarea à son contenu. Combinez avec resize="none" pour un comportement propre.

Documentation API

Le composant psh-textarea utilise des model inputs et des input signals conformes aux standards Angular 20. Le signal value sert de source de vérité unique aux deux contrats Forms.

Propriétés (Model Inputs)

Propriétés à liaison bidirectionnelle ([(ngModel)], [(value)], [formControl]).

Propriété Type Défaut Description
value string '' Valeur courante du textarea (source de vérité partagée CVA / Signal Forms)
disabled boolean false Désactive le champ et empêche toute interaction
readonly boolean false Champ visible mais non modifiable

Bonnes Pratiques d'Accessibilité

  • Label obligatoire : label est déclaré via input.required() pour forcer une description explicite à chaque usage
  • Messages d'erreur : Le conteneur d'erreur utilise role="alert" pour annoncer immédiatement les échecs de validation
  • Attributs ARIA : aria-invalid, aria-required et aria-describedby sont gérés automatiquement (helper, compteur, erreur)
  • Compteur live : Le compteur de caractères utilise aria-live="polite" pour être annoncé sans interrompre l'utilisateur
  • Focus visible : Anneau de focus conforme WCAG 2.1 AA (contraste minimum 3:1)
  • Support clavier : Navigation native via Tab, Shift+Tab, et édition standard
  • Label cliquable : Cliquer sur le label donne le focus au champ (association for / id)
  • Placeholder non substitutif : Le placeholder complète le label mais ne le remplace jamais
  • Redimensionnement : L'utilisateur conserve la possibilité de redimensionner manuellement sauf si autoSize est activé
