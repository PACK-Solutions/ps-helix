Positions du Label

Les radio buttons permettent de positionner le label à droite ou à gauche du contrôle pour s'adapter à différentes configurations de mise en page.

Label à droite (standard) Option 2

Label à Droite (Standard)

Position standard avec le label placé à droite du contrôle radio. C'est le comportement par défaut et le plus couramment utilisé.

Label à gauche Option 2

Label à Gauche

Position alternative avec le label placé à gauche du contrôle. Utile pour certaines mises en page spécifiques ou alignements particuliers.

Sans Label Visible

Radio button sans contenu projeté. Assurez-vous toujours de fournir un ariaLabel pour maintenir l'accessibilité.

États de Radio Button

Les radio buttons supportent plusieurs états pour communiquer différentes informations visuelles et contextuelles aux utilisateurs.

Option désactivée cochée Option désactivée

Désactivé

État désactivé pour les options non disponibles ou non modifiables. L'option reste visible mais non interactive.

Option avec erreur Autre option

Erreur

État d'erreur avec message pour indiquer une validation échouée ou un choix requis non effectué.

Option validée Autre option

Succès

État de succès avec message pour confirmer une sélection valide et acceptée par le système.

Option requise Autre option

Requis

État requis pour indiquer qu'une sélection dans ce groupe est obligatoire. Utiliser avec la validation de formulaire.

Tailles Disponibles

Petit radio Option 2

16px × 16px size="small"

Radio moyen Option 2

20px × 20px size="medium"

Taille par défaut, équilibrée pour la plupart des cas d'usage et une bonne lisibilité dans les formulaires standards.

Large

24px × 24px size="large"

Taille emphase pour les éléments importants nécessitant une grande visibilité ou pour les interfaces tactiles et mobiles.

Documentation API

Le composant Radio utilise des inputs bidirectionnels pour une synchronisation de l'état, conforme aux standards Angular 21.

Inputs Bidirectionnels (avec [()])

Propriété Type Défaut Description
checked boolean false État coché du radio. Utiliser avec [(checked)]
disabled boolean false État désactivé du radio

Propriétés (Input Signals)

Propriété Type Défaut Description
required boolean false Indique si le radio est requis
label string '' Label du radio (alternative au contenu projeté)
name string '' Nom du groupe radio (OBLIGATOIRE pour le groupement)
value any undefined Valeur associée au radio
error string '' Message d'erreur à afficher
success string '' Message de succès à afficher
ariaLabel string | undefined undefined Label ARIA personnalisé pour l'accessibilité
size RadioSize 'medium' 'small' | 'medium' | 'large'
labelPosition 'left' | 'right' 'right' Position du label par rapport au radio

Outputs (Événements)

Événement Type Description
checkedChange OutputEmitterRef<boolean> Émis lors du changement d'état coché
valueChange OutputEmitterRef<any> Émis avec la valeur du radio sélectionné

Exemple d'Utilisation Complète

import { PshRadioComponent } from 'ps-helix';

@Component({
  standalone: true,
  imports: [PshRadioComponent],
  template: `
    <!-- Groupe de radios pour choix de livraison -->
    <div class="radio-group">
      <h3>Mode de livraison</h3>

      <psh-radio
        name="delivery"
        value="standard"
        [(checked)]="deliveryMethod"
        [required]="true"
        size="medium"
        (valueChange)="handleDeliveryChange($event)"
      >
        Livraison standard (3-5 jours)
      </psh-radio>

      <psh-radio
        name="delivery"
        value="express"
        size="medium"
      >
        Livraison express (24h)
      </psh-radio>

      <psh-radio
        name="delivery"
        value="pickup"
        size="medium"
      >
        Retrait en magasin
      </psh-radio>
    </div>
  `
})
export class MyComponent {
  deliveryMethod = 'standard';

  handleDeliveryChange(value: any): void {
    console.log('Méthode de livraison:', value);
  }
}

Bonnes Pratiques d'Accessibilité

  • Navigation clavier : Les radios supportent la navigation par Tab, Espace, Entrée et les flèches directionnelles
  • Groupement obligatoire : Tous les radios d'un groupe doivent partager le même attribut name
  • Focus visible : Un anneau de focus visible entoure le radio lors de la navigation au clavier
  • ARIA automatique : Les attributs aria-checked, aria-invalid et aria-required sont gérés automatiquement
  • Labels obligatoires : Toujours fournir un label via contenu projeté, input label ou ariaLabel
  • Messages descriptifs : Les messages d'erreur/succès sont liés au radio via des IDs uniques pour les lecteurs d'écran
  • Navigation fléchée : Dans un groupe, les flèches haut/bas et gauche/droite permettent de naviguer entre les options
  • Un seul choix : Contrairement aux checkboxes, un seul radio peut être sélectionné dans un groupe donné

Radio vs Checkbox : Quand Utiliser Quoi ?

Radio Buttons
  • ✓ Choix unique et exclusif
  • ✓ Options mutuellement exclusives
  • ✓ Sélection obligatoire dans le groupe
  • ✓ Exemples: mode de paiement, genre, taille de vêtement
Checkboxes
  • ✓ Sélections multiples possibles
  • ✓ Options indépendantes
  • ✓ Peut être coché/décoché librement
  • ✓ Exemples: préférences multiples, conditions d'utilisation
